Git Commands Explained

git init

  • This command turns a directory into an empty Git repository
  • It is the first step in creating a repository
  • Adding and committing files or directories is only possible after running this command


    • git add

  • This command adds files to the staging area for Git
  • Before a file is available to commit to a respository, it needs to be added to the Git index (staging area)
  • It's possible to add entire directories, specific files or all unstaged files


    • git commit

  • This command records changes made to the files to a local repository
  • It is considered best practice to include a message with each commit explaining the changes made in a commit. It helps to find a particular change or understanding the changes.
  • Each commit has a unique ID


    • git status

  • This command returns the current state of the repository. It will return the current working branch
  • If a file is in the staging area, but not committed, it shows with git status
  • If there are no changes it will return "nothing to commit, working directory clean"


    • git config

  • With Git, there are many configurations and settings possible. git config is how to assign these settings.
  • Two important settings are user.name and user.email. These values set what email address and name commits will be from on a local computer
  • a --global flag is used to write the settings to all repositories on a computer. Without this flag, settings will only apply to the current repository


    • git branch

  • This command is used to determine what branch the local repository is on
  • It is also used to add a new branch or delete a branch


    • git checkout

  • This command is used to switch branches and start working in a different branch


    • git merge

  • This command combines the changes from one branch to another branch. It integrates branges together


    • git remote

  • This command is used to connect a local repository with a remote repository
  • A remote repository can have any name. It is common practice to name the remote repository origin.


    • git clone

  • This command is used to create a local working copy of an existing remote repository
  • Cloning is the equivalent of git init when working with a remote repository
  • Git will create a directory locally with all files and repository history


    • git pull

  • This command is used to get the latest version of a repository
  • It pulls the changes from the remote repository to the local computer


    • git push

  • This command sends local commits to the remote repository
  • git push requires two parameters: the remote repository and the branch that the push is for
